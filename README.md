### Описание 
Данная учебная практика направлена на изучение и реализацию алгоритма DPLL для решения задачи о выполнимости формулы в конъюнктивной нормальной форме.

### Запуск

Сборка: 
```bash
make build 
```
Исполняемые файлы лежат в build/bin/

Запуск тестов: 
```bash
make test 
```

Запуск линтера: 
```bash
make lint 
```

Проверка форматирования:
```bash
clang-format --dry-run --Werror src/*.cpp src/*.hpp
```

Форматирование исходников:
```bash
make format
```

Очистка папки build:
```bash
make clean 
```

### Замеры производительности

`measure_all.sh` является единственной точкой входа для утилит замера. Скрипт
проверяет выбор эвристик `naive` и `jwh`, фиксирует частоту CPU на время запуска
и сравнивает обе эвристики на всех `.cnf`-файлах указанного датасета:

```bash
./measure_all.sh satBenchmarks/uf50-218
```

Запуск замеров для всех наборов, перечисленных в `satBenchmarks/README.md`:
```bash
make bench
```

Перед запуском соберите solver командой `make build` и установите `hyperfine`.
Результаты сохраняются в `results/` в форматах JSON и Markdown. Скрипт использует
доступные в системе Linux механизмы фиксации частоты CPU и восстанавливает их при
завершении. Если на конкретном дистрибутиве, ядре или без нужных прав это не
поддерживается, замеры продолжаются без фиксации частоты с предупреждением.

Описание включённых наборов находится в `satBenchmarks/README.md`.
